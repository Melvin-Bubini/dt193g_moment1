Information om GitHub Pages

GitHub Pages är en kostnadsfri tjänst från GitHub som låter dig publicera statiska webbplatser direkt från ett GitHub-repository. Tjänsten är särskilt användbar för personliga projekt, portföljer eller dokumentationssidor och är huvudsakligen avsedd för statiskt innehåll. Eftersom GitHub Pages är byggt för statiska webbplatser, stöder det främst front-end-språk och vissa verktyg för statiska sidor. HTML, CSS, JavaScript är de grundläggande teknikerna för statiska webbplatser och stöds fullt ut på GitHub Pages. Du kan använda vanlig JavaScript eller exempelvis React (efter att ha kompilerat med verktyg som Webpack). Eftersom GitHub Pages bara hanterar statiska filer, finns det inget stöd för back-end-kod som exempelvis PHP, Python eller Node.js. Server-skript eller databasfunktioner kan inte köras här.

För- och nackdelar

Fördelarna är att de är kostnadsfritt. Det är helt gratis att använda för både publika och privata repositories. Det är enkelt att komma igång med för att snabbt publicera en statisk webbplats. Kodstöd för statiska sidor med HTML, CSS, JavaScript och statiska sajtgeneratorer som Jekyll. Sidan uppdateras automatiskt när du pushar ändringar till ditt repository. GitHub Pages är fri från reklam, till skillnad från vissa andra gratistjänster. Några nackdelar är att det endast är statiska webbplatser. Det stöder inte server-skript eller back-end-teknologier som PHP eller Node.js, vilket begränsar funktionaliteten. Ingen möjlighet till dynamisk datalagring direkt på sidan och större projekt kan behöva mer flexibilitet än vad GitHub Pages erbjuder, särskilt om interaktivitet krävs.

Arbetssätt

För att publicera en webbplats med GitHub Pages börjar du med att skapa ett nytt repository på GitHub, vilket fungerar som en lagringsplats för webbplatsens filer. Om du vill att webbplatsen ska vara offentlig kan du skapa ett publikt repository. Privata repositories kan också publiceras via GitHub Pages, men det kräver ett GitHub Pro-konto. När repositoryt är skapat kan du ladda upp projektets filer, som HTML, CSS, JavaScript, och bilder. Filerna kan du ladda upp direkt på GitHub eller via Git om du föredrar att arbeta lokalt på din dator. För att sedan aktivera GitHub Pages går du in i repositoryts inställningar och letar upp sektionen "GitHub Pages." Här väljer du vilken branch och eventuell undermapp som GitHub ska använda som källa för din webbplats. När detta är sparat skapas en liveversion av din webbplats inom några minuter.
